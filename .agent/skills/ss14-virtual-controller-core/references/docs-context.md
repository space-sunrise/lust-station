# Docs Context (VirtualController Core)

## Роль документации

1. Docs используй для терминологии, intent и диагностики.
2. Поведение системы всегда сверяй по текущему коду.
3. При конфликте docs vs код выбирай код.

## Релевантные страницы

1. `robust-toolbox/transform/physics.md` — дата `2023-09-21`.
Назначение: общая картина physics pipeline и место VirtualControllers.
Ограничение доверия: страница устарела, есть TODO, использовать только как high-level intent.

2. `ss14-by-example/prediction-guide.md` — дата `2026-02-04`.
Назначение: практики prediction, replay/re-sim и guard-паттерны.
Ограничение доверия: ориентир по методологии, но конкретные примеры всегда сверять с текущими системами.

3. `ss14-by-example/basic-networking-and-you.md` — дата `2024-05-31`.
Назначение: `Dirty`/network fields/ComponentState для shared-предикт-сценариев.
Ограничение доверия: полезно как базовая сеть-модель, но не заменяет чтение runtime-кода.

## Практика применения docs

1. Сначала находи свежую реализацию в коде (после cutoff `2024-02-20`).
2. Затем проверяй docs на терминологию и объяснение причин.
3. Если docs толкают в legacy-подход, фиксируй это в `rejected-snippets.md`.
