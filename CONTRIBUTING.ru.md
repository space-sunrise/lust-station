# Правила контрибьютинга

## 1. Типы контента

Следующий контент открыт для публичного контрибьютинга:

* YAML
* Спрайты
* Звуки
* Карты
* Переводы
* Документация

Такие PR обычно проще и быстрее ревьюить.

---

## 2. Ограничения на C#

C# Pull Requests требуют дополнительного одобрения.

Вы можете отправлять C# PR только если:

* Вы состоите в команде разработки
  ИЛИ
* PR привязан к Issue с меткой `Одобрено`

Данная политика введена для снижения нагрузки на мейнтейнеров и поддержания качества кода.

---

## 3. Требования к Pull Request

Перед merge:

* Все тесты должны быть зелёными
* Все комментарии ревью (включая AI) должны быть закрыты
* В PR не должно оставаться открытых обсуждений

Мейнтейнеры могут переводить PR в Draft до выполнения всех требований.

---

## 4. Соответствие GDD

**GDD в разработке.**

Все изменения игрового контента должны соответствовать GDD проекта.

Контент, противоречащий дизайну сервера или репозитория, может быть отклонён.

---

## 5. Общие правила

* Делайте PR небольшими и сфокусированными
* Крупные смешанные PR могут быть отклонены
* Низкокачественный AI-код не принимается
* Поддерживаемость и читаемость кода обязательны
